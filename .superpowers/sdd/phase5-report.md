## Values rationale (capacity-fit on 2× t3.large ≈ 4 vCPU / 16 GiB)

- **kube-prometheus-stack:** alertmanager disabled. Prometheus retention 5d,
  req 200m/512Mi lim 500m/1Gi, gp2 10Gi PVC, `serviceMonitor/podMonitor
  SelectorNilUsesHelmValues:false` (picks up app ServiceMonitors in dev/staging/prod).
  `prometheusOperator.admissionWebhooks.enabled:false` (no cluster-wide validating
  webhook) **plus `prometheusOperator.tls.enabled:false`** — required because the
  operator's TLS-serving cert is generated by the admission-webhook cert job; with
  the webhook off the operator would otherwise hang on a missing
  `kube-prometheus-stack-admission` secret (this was the second install failure,
  now fixed). node-exporter pinned via the `prometheus-node-exporter` subchart key
  (nodeSelector `workload=platform`). Grafana: admin creds from TF secret
  `grafana-admin`, gp2 5Gi PVC, dashboard+datasource sidecars enabled, Loki+Tempo
  as `additionalDataSources`, Prometheus default (uid `prometheus`).
- **Loki:** SingleBinary + filesystem, replication_factor 1, tsdb schema v13,
  72h retention. `compactor.delete_request_store: filesystem` +
  `working_directory` — **required once retention_enabled** (this was the first
  install failure: "delete-request-store should be configured when retention is
  enabled"; now fixed). gateway/minio/caches/lokiCanary/test/serviceMonitor all
  disabled (lokiCanary is a DaemonSet with no nodeSelector → disabled so it can't
  land on ng-dense). gp2 10Gi PVC, nodeSelector platform.
- **Promtail:** DaemonSet nodeSelector `workload=platform`; JSON pipeline extracts
  `level`/`service`/`trace_id`; ships to loki.
- **Tempo:** single-binary, local filesystem backend (chart default), gp2 10Gi PVC,
  24h retention, OTLP receivers only (4317/4318), nodeSelector platform.
- **OTel Collector:** deployment mode, contrib image, `alternateConfig` used (full
  override) so only the OTLP receiver + `otlp/tempo` exporter + traces pipeline are
  active (chart-default jaeger/zipkin receivers and debug exporter dropped); jaeger/
  zipkin Service ports disabled → Service exposes only otlp:4317 / otlp-http:4318.
