# Phase 7 — CI/CD (GitHub Actions + SonarQube + Trivy + OIDC) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Wire the full CI/CD: GitHub OIDC → AWS role (no static keys), SonarQube project + token, GitHub repo secrets, and four GitHub Actions workflows (pr-checks, ci, release, terraform) with SonarQube quality gate + Trivy scanning + image build/push to ECR + cosign signing. Prove it by triggering a real run and watching it pass.

**Architecture:** GitHub Actions authenticate to AWS via OIDC (a Terraform-managed IAM role trusted for the repo). `pr-checks` runs on PRs (lint/test per changed service, Sonar scan blocking on quality gate, Trivy fs/config). `ci` runs on push to main (build changed images, Trivy image scan, cosign sign, push to ECR, then `terraform apply` to roll the new tag into dev). `release` runs on `v*` tags (promote staging→prod with manual approval). `terraform` runs fmt/validate/tflint/plan on PRs touching `terraform/`. SonarQube is the Phase 4 EC2 (`http://13.200.32.13:9000`).

**Tech Stack:** GitHub Actions, GitHub OIDC, AWS IAM, SonarQube Community, Trivy (aquasecurity/trivy-action), cosign, docker buildx, Terraform, `gh` CLI, actionlint.

### Task 4: GitHub Actions workflows

**Files:** `.github/workflows/{pr-checks,ci,release,terraform}.yml`.

**Interfaces:** the four pipelines. `permissions: id-token: write, contents: read` on jobs using OIDC.

- [ ] **Step 1: `pr-checks.yml`** — on `pull_request` to main. Jobs (path-filtered via `dorny/paths-filter` per service): per changed service → setup toolchain → lint + unit test (Go via container/setup-go, Node via setup-node, Python via setup-python) → produce coverage. A `sonar` job: SonarQube scan (`sonarsource/sonarqube-scan-action` or the CLI) against `SONAR_HOST_URL`/`SONAR_TOKEN` with `-Dsonar.qualitygate.wait=true` (BLOCKS on quality gate). A `trivy` job: `aquasecurity/trivy-action` fs scan + config/IaC scan, `exit-code: 1` on HIGH/CRITICAL, upload SARIF.
- [ ] **Step 2: `ci.yml`** — on `push` to main. `permissions: id-token: write`. Jobs: build changed images (buildx) → Trivy **image** scan (fail HIGH/CRITICAL) → cosign sign (keyless/OIDC) + SBOM → `aws-actions/configure-aws-credentials` (OIDC, role `AWS_DEPLOY_ROLE_ARN`) → push to ECR tag `sha-${GITHUB_SHA::7}` → deploy job: `terraform -chdir=terraform/stack apply -auto-approve -var app_image_tag=sha-${SHA}` (rolls the new image into dev). Guard deploy behind success of scans.
- [ ] **Step 3: `release.yml`** — on push tag `v*`. Promote the already-built image: deploy to `staging` (helm/terraform with the tag) then a `prod` job gated by a GitHub **Environment** `production` requiring manual approval. (Uses OIDC role.)
- [ ] **Step 4: `terraform.yml`** — on `pull_request` touching `terraform/**`: `fmt -check`, `validate`, `tflint`, `plan` (read-only, OIDC role with plan perms) posting the plan; on `workflow_dispatch`: `apply`.
- [ ] **Step 5: Lint workflows** — run `actionlint` (via docker `rhysd/actionlint`) on all four → 0 errors.
- [ ] **Step 6: Commit** — `feat(ci): GitHub Actions pipelines (pr-checks, ci, release, terraform)`.
