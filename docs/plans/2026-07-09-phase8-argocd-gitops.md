# Phase 8 — ArgoCD GitOps — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Introduce ArgoCD as the GitOps deployer for the app. ArgoCD (installed by Terraform) watches the git repo and syncs the app Helm chart into dev/staging/prod. Terraform stops directly deploying the app; CI's deploy step becomes "commit the new image tag to git → ArgoCD auto-syncs." Prove the full GitOps loop, then the app is deployed purely from git.

**Architecture:** Terraform installs ArgoCD (helm_release, `argocd` namespace, platform nodes) and a single root **app-of-apps** `Application` that points at `deploy/argocd/apps/`. That directory holds per-environment `Application`s (dev/staging/prod), each sourcing the in-repo Helm chart `deploy/helm/ztd-capstone` with the matching `values-<env>.yaml`. The `dev` Application uses automated sync + self-heal; staging/prod are manual sync. The image tag lives in git (`values-dev.yaml` `image.tag`); CI bumps it and commits, and ArgoCD rolls it out. Terraform keeps owning the Postgres Secret (not in git) and ArgoCD; it no longer owns the app workloads.

**Tech Stack:** ArgoCD (argo-cd Helm chart), Terraform helm/kubernetes providers, the Phase 3 app chart, GitHub Actions, public GitHub repo (no ArgoCD repo creds needed).
